2026-04-22 · Sprint 2 code complete — paste-an-address → verdict pipeline

First functional product surface. User pastes a US address on
/app/properties, Scout runs Anthropic Sonnet 4.6 with web_search
(up to 8 calls), renders a BUY/WATCH/PASS verdict in 20-40s.

Shipped (M5-M8):
  - Schema: properties.google_place_id, properties.address_full,
    verdicts table (immutable snapshots per property), user_verdict_limits
    (free-tier metering: 3/30-day window, lazy reset)
  - AddressAutocomplete: Google Places JS API v2 functional loader,
    US + street-level only, Zod validation of parsed shape
  - Server action createPropertyAction: auth → user/org resolve →
    address validate → property upsert (dedupe on place_id then
    normalized address) → free-tier consume → pending verdict row
  - packages/ai: task registry entry for verdict-generation, prompt
    loaded from prompts/verdict-generation.v1.md with fair-housing
    safety instruction, Zod output schema cross-validates tool call
  - Route handler POST /api/verdicts/[id]/generate: Node runtime,
    maxDuration 60s, idempotent (already-ready short-circuits; failed
    retries overwrite), refunds quota slot on failure
  - Property detail page: VerdictCertificate renders placeholder /
    pending / ready / failed states; VerdictLoader client component
    auto-triggers generation on mount then router.refresh()

Observability: every verdict logs model_version, prompt_version,
input_tokens, output_tokens, cost_cents per CLAUDE.md AI non-negotiable.
Pricing table in packages/ai/src/pricing.ts matches 2026-04 Anthropic
sheet. PostHog / Sentry / Axiom wiring deferred to Sprint 3 (ADR-4).

Deviation from CLAUDE.md logged as ADR-3: verdict generation lives
in a route handler, not a server action, because 20-40s blocking
doesn't compose with the create-then-redirect UX the client needs.

Tests: 9 unit in apps/web (address parsing + normalization) and 11
in packages/ai (pricing + VerdictOutputSchema golden fixtures).
clerk-sync integration tests self-skip when DATABASE_URL is unset
(describe.skipIf) so unit runs work standalone. Fair-housing golden
file suite deferred to Sprint 4 per CLAUDE.md (lands with Location
Signals).

New dependencies (each tied to a specific milestone):
  - @googlemaps/js-api-loader + @types/google.maps (M5)
  - server-only (M5; defense-in-depth boundary on lib/db/queries/)
  - @anthropic-ai/sdk in packages/ai + apps/web (M7)
  - zod in packages/ai (M7; matches apps/web's version)

Blocked on keys for M9 (deploy + smoke): NEXT_PUBLIC_GOOGLE_MAPS_API_KEY
and ANTHROPIC_API_KEY in Vercel, plus DATABASE_URL locally to apply
migration 0001 against the Neon dev branch. Runbook written at
docs/RUNBOOKS/sprint-2-verdict-generation.md.
